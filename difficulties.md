# Порівняльний аналіз алгоритмів детермінізації НКА

## Позначення

| Символ | Значення |
|--------|----------|
| **n** | Кількість станів НКА |
| **m** | Кількість переходів НКА |
| **\|Σ\|** | Розмір алфавіту |
| **D** | Кількість досяжних станів результуючого ДКА (worst-case: 2^n) |
| **D_min** | Кількість станів мінімального ДКА |
| **k** | Кількість singularities (недетермінованих переходів) у НКА |
| **D'** | Кількість станів ДКА після злиття еквівалентних (D' <= D) |
| **d** | Щільність переходів: m / (n * \|Σ\| * n) |
| **nd** | Ступінь недетермінізму: частка переходів з >1 ціллю |
| **da(w)** | Ступінь неоднозначності на слові w: кількість приймальних шляхів |
| **T_lazy** | Кількість фактично обчислених переходів при lazy-підході |

## 1. Основна таблиця складностей

| Параметр | Subset Construction | Brzozowski | Transset | Quick Subset (QSC) | Lazy Subset |
|----------|-------------------|------------|----------|-------------------|-------------|
| **Часова складність (worst)** | O(2^n * n * \|Σ\|) | O(2^(2^n) * \|Σ\|) | O(2^n * m * \|Σ\|) | O(2^n * n * \|Σ\|) | O(2^n * n * \|Σ\|) |
| **Часова складність (typical)** | O(D * n * \|Σ\|) | O(D₁ * n * \|Σ\| + D₂ * D₁ * \|Σ\|) | O(D' * m * \|Σ\|) | O(n * \|Σ\| + k * D * n * \|Σ\|) | O(D * n * \|Σ\|) |
| **Часова складність (best)** | O(n * \|Σ\|) (якщо НКА вже дет.) | O(4 * n * \|Σ\|) (4 проходи) | O(n * \|Σ\|) | **O(n * \|Σ\|)** (0 singularities) | O(n * \|Σ\|) |
| **Часова складність (average)** | O(D_avg * n * \|Σ\|) | Суперполіноміальна навіть у середньому (доведено для \|Σ\| >= 2, De Felice & Nicaud, 2013; результат стосується мінімізації ДКА) | O(D'_avg * m * \|Σ\|) | Залежить від nd | O(D_avg * n * \|Σ\|) |
| **Просторова складність (worst)** | O(2^n * n + 2^n * \|Σ\|) | O(2^(2^n)) | O(2^n * n + 2^n * \|Σ\|) | O(2^n * n + n * \|Σ\|) | O(2^n * n + 2^n * \|Σ\|) |
| **Просторова складність (typical)** | O(D * n + D * \|Σ\|) | O(D₁ * n + D₂ * D₁) | O(D' * n + D' * \|Σ\|) | O(n * \|Σ\| + D * n) | O(D * n + D * \|Σ\|) |
| **Пікова пам'ять під час роботи** | O(D * n) (черга підмножин) | O(D₁ * n + D₂ * D₁) (кілька проміжних автоматів одночасно в пам'яті) | O(D' * n) (черга кортежів) | O(n * \|Σ\| + D * n) (клон + merged) | O(D * n) (кеш + стек) |
| **Кількість проходів по НКА** | 1 (один BFS) | 4 (reverse → det → reverse → det) | 1 (один BFS) | 3 фази (clone → restructure → cleanup) | 1 (один DFS) |
| **Вартість одного кроку BFS** | O(n) — об'єднання множин | O(n) — об'єднання множин | O(\|T_current\|) — сканування transition set поточного стану | O(n) — nfa_move по extension | O(n) — об'єднання множин (з кешуванням) |

| **Сильна зв'язність компонент** | Може змінитися | Оптимальна | Може змінитися | Близька до НКА | Може змінитися (як Subset) |

## 4. Міри недетермінізму та неоднозначності НКА

> Ці метрики характеризують вхідний НКА і впливають на продуктивність кожного алгоритму.

| Метрика | Визначення | Вплив на Subset | Вплив на Brzozowski | Вплив на Transset | Вплив на QSC | Вплив на Lazy |
|---------|-----------|----------------|--------------------|--------------------|-------------|---------------|
| **Ступінь недетермінізму (nd)** | Частка переходів (q, a) де \|δ(q, a)\| > 1 | Не враховує | Не враховує | Частково враховує | **Ключовий**: час ~ O(nd * D) | Не враховує |
| **Ступінь неоднозначності (ambiguity degree)** | max по всіх словах w кількості приймальних шляхів da(w); для скінченно-неоднозначних НКА: da(w) <= 5n * 2^(n²) (Weber & Seidl, 1991) | Не впливає на алгоритм | Не впливає на алгоритм | Не впливає на алгоритм | Не впливає на алгоритм | Не впливає на алгоритм |
| **Обмежена неоднозначність** | Існує k: для будь-якого w, da(w) <= k | D зазвичай менше | Brz ефективніший | D' значно менше | QSC ефективний | D зазвичай менше |
| **Ширина дерева обчислень (tree width)** | Max кількість активних гілок на одному кроці | D ~ 2^(tree_width) | Аналогічно | D' < D | Не впливає напряму | D ~ 2^(tree_width) |
| **Середній branching factor** | Середнє \|δ(q, a)\| для непустих переходів | Визначає швидкість росту BFS | Визначає розмір reverse | Визначає кількість злиттів | = singularity density | Визначає швидкість росту DFS |
| **Циклічність (cyclicity index)** | Довжина найдовшого циклу в графі НКА | Впливає на subset complexity | 2× вплив (подвійна детермінізація) | Аналогічно Subset | Мінімальний вплив | Аналогічно Subset |
| **Ранг матриці переходів** | Ранг булевої матриці M_a для кожного символу a | Верхня межа subset complexity | Подвійний вплив | Аналогічний | Мінімальний вплив | Аналогічно Subset |

### Класифікація НКА за неоднозначністю

| Клас НКА | Визначення | Верхня межа D | Найкращий алгоритм |
|----------|-----------|---------------|-------------------|
| **Детерміновані** | nd = 0, всі переходи однозначні | D = n | QSC (просто копіює) |
| **Однозначні (UFA)** | Для кожного w рівно 0 або 1 приймальний шлях | D <= 2^n, але зазвичай << 2^n | Subset, QSC або Lazy |
| **Обмежено неоднозначні** | da(w) <= k для фіксованого k | D <= 2^n (верхня межа не знижується автоматично, але на практиці D значно менше) | Transset (зливає стани) |
| **Поліноміально неоднозначні** | da(w) <= p(n) для полінома p | D <= 2^n (теоретично), на практиці зазвичай поліноміальний | Будь-який |
| **Експоненційно неоднозначні** | da(w) може бути 2^n | D = 2^n (worst-case) | Brz (мінімізує) |

## 5. Subset Complexity та прогнозування вибуху станів

> Subset complexity (SC) — метрика, що дає верхню межу для розміру ДКА, побудованого subset construction. Обчислюється через ранг та циклічність матриць переходів (Baburin & Cotterell, "A Close Analysis of the Subset Construction", 2024).

| Параметр | Опис | Формула / межа |
|----------|------|---------------|
| **Subset complexity** | Верхня межа кількості досяжних підмножин | Визначається через комбінацію range bound та monoid bound; практична апроксимація використовує ранг та циклічність (точна формула складна, див. Definition 9 у статті) |
| **Ранг матриці переходів M_a** | Булева матриця n×n над F₂: M_a[i][j] = 1 якщо j ∈ δ(i, a) | 1 <= rank(M_a) <= n; входить в апроксимацію як 2^(rank²/4) |
| **Циклічність (cyclicity)** | Циклічність графу переходів для обраного символу | Входить в апроксимацію SC лінійно |
| **Tight upper bound** | 2^n (Rabin-Scott, 1959) | Досягається на сімействі "n-й символ з кінця" |
| **Tight lower bound для UFA** | 2^n (навіть однозначні НКА можуть давати вибух) | Доведено окремими конструкціями |
| **Обчислювальна складність оцінки** | PSPACE-hard визначити, чи буде вибух | Неможливо передбачити точно за поліноміальний час |

### Як subset complexity впливає на алгоритми

| Алгоритм | Зв'язок з subset complexity |
|----------|---------------------------|
| **Subset** | Генерує рівно стільки станів, скільки досяжних підмножин. SC — точна верхня межа |
| **Brzozowski** | Перша детермінізація обмежена SC reverse автомата, друга — SC результату |
| **Transset** | Зливає підмножини з однаковою transition set → менше або дорівнює SC |
| **QSC** | Генерує ті ж підмножини що й Subset → обмежений тією ж SC |
| **Lazy** | Генерує ті ж підмножини що й Subset (DFS замість BFS) → обмежений тією ж SC |

## 7. Практичні характеристики реалізації

| Параметр | Subset | Brzozowski | Transset | QSC | Lazy Subset |
|----------|--------|------------|----------|-----|-------------|
| **Простота реалізації** | Найпростіший (~40 рядків) | Середня (~80 рядків, 4 кроки) | Середня (~55 рядків) | Найскладніший (~140 рядків, 3 фази) | Проста (~100 рядків, кеш + DFS) |
| **Кількість допоміжних структур** | 1 (subset_to_name) | 4+ (2 DFA + 2 reversals) | 2 (trans_by_state + tuple_to_name) | 3 (ext_to_name, trans, needs_transitions) | 2 (cache + subset_to_name) |
| **Тип основної структури даних** | dict: frozenset → name | dict: frozenset → name (×2) | dict: (frozenset, bool) → name | dict: frozenset → name + set-based trans | dict: (frozenset, symbol) → frozenset (кеш) |
| **Кеш-локальність (cache locality)** | Середня (хеш-таблиці) | Погана (4 проходи, різні структури) | Середня | Хороша для дет. частини, середня для merged | Середня (хеш-таблиця кешу) |
| **Придатність для інкрементальної обробки** | Ні (повна перебудова) | Ні (повна перебудова) | Ні (повна перебудова) | Теоретично так (алгоритм дозволяє дообробку нових singularities), але поточна реалізація — ні | Теоретично так (кеш можна розширювати), поточна реалізація — ні |
| **Паралелізація** | BFS рівнями (обмежена) | Послідовні 4 фази (неможлива) | BFS рівнями (обмежена) | Фази 1-2 незалежні від фази 3 | Обмежена (DFS послідовний) |
| **Пам'ять під час виконання** | Черга підмножин | 2 проміжні автомати одночасно | Черга кортежів | Клон НКА + нові стани | Стек + кеш переходів |
| **Overhead при малому НКА (n < 10)** | Мінімальний | Високий (4 проходи) | Середній | Середній (3 фази) | Мінімальний |
| **Стабільність часу виконання** | Стабільний (детермінований BFS) | Нестабільний (залежить від reverse) | Стабільний | Залежить від nd | Стабільний (детермінований DFS) |
| **Garbage collection pressure** | Середній (створення frozenset) | Високий (проміжні автомати) | Середній | Низький при sparse, високий при dense | Середній (створення frozenset + кеш) |

## 8. Порівняння поведінки на класичних сімействах НКА

| Сімейство НКА | n | D (Subset/QSC/Lazy) | D_min (Brz) | D' (Transset) | Відомий результат |
|---------------|---|---------------------|-------------|---------------|-------------------|
| **n-й символ з кінця** | n+1 | 2^n | 2^n | 2^n | Tight bound, всі алгоритми дають однаково |
| **Об'єднання n мов a^i** | n+1 | n+1 | n+1 | n+1 | Лінійний, вибуху немає |
| **k паралельних гілок довжини n** | k*n+2 | O(n^k) | < n^k | <= n^k | Декартовий добуток позицій |
| **(a\|b)* a (a\|b)^n** | n+2 | 2^(n+1) | 2^(n+1) | 2^(n+1) | Варіант nth-from-last |
| **Випадковий dense (p=0.3)** | n | O(2^n) | << 2^n | < 2^n | В середньому значно менше worst-case |
| **Випадковий sparse (nd=5%)** | n | ≈ n | ≈ n | ≈ n | DFA ≈ NFA розміру |
| **Ланцюговий (chain)** | n | n | n | n | Вже детермінований |
| **Зірка Кліні a*\|b*** | 3 | 4 | 3 | 3-4 | Мінімальний виграш від Brz |

## 9. Найкращі та найгірші сценарії

| Сценарій | Найкращий алгоритм | Найгірший алгоритм | Причина |
|----------|-------------------|-------------------|---------|
| **Worst-case (n-й символ з кінця)** | Subset або Lazy (базовий) | Brzozowski (подвійна детермінізація 2^n станів) | Всі дають 2^n станів, Brz робить зайву роботу |
| **Щільний випадковий НКА (d > 0.3)** | Brzozowski | QSC | Brz дає мінімальний DFA; QSC не має виграшу — все недетерміноване |
| **Розріджений НКА (5% nondet)** | **QSC** | Brzozowski | QSC зберігає 95% без змін; Brz робить повну перебудову |
| **Симетричні гілки** | Transset / Brzozowski | Subset / Lazy | Transset зливає симетричні стани; Brz мінімізує |
| **Вже детермінований автомат** | **QSC** (O(n * \|Σ\|)) | Brzozowski | QSC: 0 singularities, просто копіює; Brz: 4 проходи даремно |
| **Потрібен мінімальний ДКА** | Brzozowski | Subset / Lazy | Тільки Brz гарантує мінімальність |
| **Обмежена пам'ять** | Transset (D' < D) | Brzozowski (2 проміжні ДКА) | Transset зменшує кількість станів |
| **Великий алфавіт (\|Σ\| > 10)** | QSC (при sparse) | Brzozowski | Множник \|Σ\| × 4 проходи |
| **НКА з epsilon-переходами** | Subset або Lazy (простий epsilon-closure) | Brzozowski (epsilon-closure × 4) | Brz множить overhead на 4 |
| **Інкрементальне додавання станів** | QSC або Lazy (теоретично підтримують, в поточній реалізації — ні) | Всі (повна перебудова) | Кеш Lazy та архітектура QSC дозволяють дообробку |
| **Однозначний НКА (UFA)** | QSC, Subset або Lazy | Brzozowski | UFA мають менший D, Brz робить зайву роботу |
| **Циклічний НКА (багато циклів)** | Subset або Lazy | Brzozowski | Цикли збільшують subset complexity, Brz страждає вдвічі |

## 12. Формули зв'язності графу

### До детермінізації (НКА)

- **Ступінь вершини (out-degree):** `deg(q) = |{(q, a, q') ∈ δ}|` для всіх символів і цілей
- **Середній out-degree:** `avg_deg = m / n`
- **Максимальний out-degree:** `max_deg = max_q |{(q, a, q') ∈ δ}|`
- **Щільність переходів:** `d = m / (n × |Σ| × n)` (частка від максимально можливих переходів)
- **Ступінь недетермінізму:** `nd = |{(q, a) : |δ(q, a)| > 1}| / |{(q, a) : δ(q, a) ≠ ∅}|`
- **Середній branching factor:** `bf = Σ |δ(q, a)| / |{(q, a) : δ(q, a) ≠ ∅}|` для всіх непустих переходів
- **In-degree:** `in_deg(q) = |{(p, a) : q ∈ δ(p, a)}|`
- **Середній in-degree:** `avg_in_deg = m / n` (рівний out-degree)
- **Кількість компонент сильної зв'язності:** SCC count
- **Діаметр графу:** максимальна відстань між парою досяжних вершин
- **Циклічність:** довжина найдовшого простого циклу

### Після детермінізації (ДКА)

- **Ступінь вершини:** завжди `|Σ|` (повністю визначений, кожен стан має перехід по кожному символу)
- **Щільність:** завжди 1.0 (повний граф по алфавіту)
- **Коефіцієнт вибуху:** `blowup = D / n`
- **Коефіцієнт стиснення (Brz vs Subset):** `compression = D_min / D`
- **Коефіцієнт стиснення (Transset vs Subset):** `compression_ts = D' / D`
- **Ефективність мінімізації:** `efficiency = (D - D_min) / D × 100%`

### Зміна зв'язності при детермінізації

| Метрика | НКА | ДКА | Зміна |
|---------|-----|-----|-------|
| Кількість станів | n | D (до 2^n) | Зростає експоненційно (worst) |
| Кількість переходів | m | D × \|Σ\| | Зростає (повний граф) |
| Середній out-degree | m / n | \|Σ\| | Стає фіксованим |
| Максимальний out-degree | до n × \|Σ\| | \|Σ\| | Стає фіксованим |
| Недетермінізм (nd) | nd > 0 | 0 | Зникає повністю |
| Branching factor | bf > 1 | 1 | Стає одиничним |
| Досяжність | Може мати недосяжні | Всі досяжні | BFS гарантує |
| "Мертві" стани | Можливі | Можливі (крім Brz) | Потребує окремого видалення |
| Діаметр | d_NFA | d_DFA >= d_NFA | Може зрости |
| Компоненти зв'язності | >= 1 | 1 (один досяжний компонент) | Зливаються |
| Неоднозначність | da(w) >= 0 | Завжди 0 або 1 | Зникає повністю |

## 13. Порівняння overhead для epsilon-переходів

| Параметр | Subset + ε | Brzozowski + ε | Transset + ε | QSC + ε | Lazy + ε |
|----------|-----------|---------------|-------------|---------|----------|
| **Додатковий крок** | ε-closure перед кожним move | ε-closure при першій детермінізації (reverse НКА з ε); друга детермінізація вже без ε (результат першої — ДКА) | ε-closure перед кожним move | ε-closure перед кожним nfa_move | ε-closure перед кешуванням кожного переходу |
| **Складність ε-closure** | O(n²) (BFS по ε-переходах) | O(n²) для першої детермінізації; друга детермінізація без ε-overhead | O(n²) | O(n²) | O(n²) |
| **Кешування ε-closure** | Можливе (обчислити раз) | Можливе для першої фази | Можливе | Можливе | Вбудоване (кеш переходів вже включає ε-closure) |
| **Збільшення часу** | +30-50% (типово) | +15-30% (ε тільки на першому з 4 проходів) | +30-50% | +30-50% (лише для singularities) | +30-50% |
| **Збільшення пам'яті** | +O(n²) для таблиці closure | +O(n²) | +O(n²) | +O(n²) | +O(n²) |
| **Вплив на розмір ДКА** | Може зменшити (ε зливає стани) | Автоматично мінімізує | Може зменшити | Може зменшити | Може зменшити |
