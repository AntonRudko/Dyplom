# Helpers — допоміжні модулі для юніт-тестів

Папка `helpers/` містить 4 модулі, які використовуються тестами для запуску автоматів, генерації слів, перевірки еквівалентності та мінімізації ДКА.

---

## dfa_helpers.py — запуск автоматів та генерація слів

### `run_dfa(dfa, word) → bool`

Запускає слово на ДКА. Починає зі стартового стану, послідовно переходить за символами слова. Повертає `True`, якщо кінцевий стан — приймаючий. Якщо перехід не визначений — повертає `False`.

### `run_nfa(nfa, word) → bool`

Запускає слово на НКА з підтримкою ε-переходів. Використовує множину поточних станів та ε-замикання (`_epsilon_closure`) після кожного символу. Повертає `True`, якщо хоча б один кінцевий стан — приймаючий.

### `gen_random_words(alphabet, count=200, max_len=15) → list[str]`

Генерує `count` випадкових слів довжиною від 0 до `max_len` над заданим алфавітом. Завжди включає порожнє слово `""`. Символ ε (`""`) виключається з алфавіту.

### `gen_systematic_words(alphabet, max_len=4) → list[str]`

Генерує **всі** можливі слова довжиною від 0 до `max_len` (декартів добуток). Використовується для вичерпної перевірки на малих автоматах.

## equivalence.py — перевірка еквівалентності мов

### `check_language_equivalence_by_words(nfa, dfa, count=300, max_len=15) → (bool, list)`

Перевіряє, що ДКА приймає ту ж мову, що й НКА. Генерує 300+ випадкових слів + всі систематичні слова до довжини 3. Порівнює результат `run_nfa` та `run_dfa` для кожного слова.

- Повертає `(True, [])` якщо розбіжностей немає
- Повертає `(False, [(слово, nfa_result, dfa_result), ...])` при знайдених розбіжностях

### `check_dfa_equivalence_by_minimization(dfa1, dfa2) → bool`

Перевіряє еквівалентність двох ДКА через мінімізацію обох та перевірку ізоморфізму мінімальних автоматів. Це найсильніша форма перевірки — структурна еквівалентність.

### `check_all_algorithms_equivalent(nfa, algorithms) → (bool, dict)`

Комплексна перевірка: запускає всі алгоритми на одному НКА, перевіряє:
1. Мовну еквівалентність кожного ДКА з НКА (словами)
2. Попарну ізоморфність мінімізованих ДКА

Параметри:
- `nfa` — вхідний НКА
- `algorithms` — список пар `(назва, функція)`, де `функція(nfa) → (DFA, int)`

Повертає `(all_ok, results_dict)` з детальними результатами по кожному алгоритму.

## minimization.py — мінімізація ДКА (алгоритм Хопкрофта)

### `minimize_dfa(dfa) → MinimalDFA`

Мінімізує ДКА методом partition refinement (стиль Хопкрофта). Етапи:

1. **BFS для досяжних станів** — видаляє недосяжні стани
2. **Додавання dead state** — робить автомат повним (тотальним)
3. **Partition refinement** — початковий розбиток `{accept, non-accept}`, потім ітеративне розщеплення за сигнатурами переходів до стабільності
4. **Побудова мінімального ДКА** — кожен клас еквівалентності → один стан
5. **Видалення dead state** — повертає частковий ДКА (сумісний з виходом Subset/Transset)

### `class MinimalDFA`

Легковісний контейнер для мінімізованого ДКА з полями: `states`, `alphabet`, `transitions`, `start_state`, `accept_states`.

## isomorphism.py — перевірка ізоморфізму ДКА

### `are_isomorphic(dfa1, dfa2) → bool`

Перевіряє, чи два ДКА ізоморфні — тобто чи існує бієкція між їхніми станами, що зберігає переходи та прийняття.

Алгоритм:
1. Швидкі перевірки: однаковий алфавіт, кількість станів, кількість приймаючих станів
2. BFS-конструкція бієкції починаючи зі стартових станів
3. Для кожного символу: якщо цільові стани вже відображені — перевіряє консистентність; якщо ні — додає нову пару до відображення
4. Перевіряє, що всі досяжні стани покриті бієкцією

Два мінімальних ДКА, що розпізнають одну мову, завжди ізоморфні — це ключова властивість, на якій базується `check_dfa_equivalence_by_minimization`.
