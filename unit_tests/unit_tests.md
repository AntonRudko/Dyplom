# Unit Tests — Інструкція

## Що це таке

Папка `unit_tests` містить **62 тести** для перевірки коректності **детермінізації НКА → ДКА** чотирма алгоритмами: **Subset Construction, Brzozowski, Transset, Lazy Subset** (+ їхні epsilon-варіанти).

---

## Структура

```
unit_tests/
├── helpers/
│   ├── dfa_helpers.py      — запуск ДКА/НКА на словах, генерація тестових слів
│   ├── equivalence.py      — перевірка еквівалентності мов (словами + ізоморфізмом)
│   ├── isomorphism.py      — перевірка ізоморфізму двох ДКА (BFS-біекція)
│   └── minimization.py     — мінімізація ДКА (алгоритм Хопкрофта)
├── test_subset.py           — 9 тестів Subset Construction
├── test_brzozowski.py       — 8 тестів Brzozowski
├── test_transset.py         — 8 тестів Transset
├── test_lazy.py             — 10 тестів Lazy Subset
├── test_epsilon_variants.py — 5 тестів epsilon-варіантів (всі 4 алгоритми)
├── test_cross_algorithm.py  — 8 тестів крос-алгоритмової еквівалентності (КЛЮЧОВИЙ)
├── test_edge_cases.py       — 8 тестів граничних випадків (всі 4 алгоритми)
└── test_minimization.py     — 6 тестів мінімізації та ізоморфізму
```

---

## Які перевірки виконуються

Кожен алгоритм перевіряється на 5 рівнях:

| Рівень | Що перевіряється | Як |
|--------|-------------------|----|
| **1. Структура** | ДКА має валідні стани, початковий стан ∈ states, accept ⊆ states | Прямі assertions |
| **2. Детермінізм** | Кожна пара (стан, символ) → рівно ОДИН стан | Перевірка що target не list/set |
| **3. Мовна еквівалентність** | ДКА приймає ту саму мову що НКА | 300+ випадкових слів + систематичні слова до довжини 3-5 |
| **4. Крос-алгоритмова еквівалентність** | Всі 4 алгоритми дають однаковий результат | Порівняння прийняття слів між усіма парами ДКА |
| **5. Структурна еквівалентність** | Мінімізовані ДКА ізоморфні | Мінімізація + BFS-біекція |

## Як зрозуміти що детермінізація пройшла коректно

### Запуск усіх тестів

```bash
cd Dyplom_Top
python3 -m unittest discover -s unit_tests -p "test_*.py" -v
```

### Результат при УСПІХУ

```
test_returns_dfa_and_count (unit_tests.test_subset.TestSubsetConstruction) ... ok
test_dfa_is_deterministic (unit_tests.test_subset.TestSubsetConstruction) ... ok
test_language_equivalence_predefined_nfa1 ... ok
...
Ran 62 tests in 50s

OK    ← ← ← Все коректно!
```

### Результат при ПОМИЛЦІ

```
FAIL: test_language_equivalence_predefined_nfa1
AssertionError: Mismatches: [('aab', True, False), ('ba', False, True)]
                              слово   НКА    ДКА
```

Це означає: слово `'aab'` НКА приймає, а ДКА — ні (або навпаки). Детермінізація **некоректна**.

---

## Ключові індикатори коректності

| Що дивитися | Значення |
|-------------|----------|
| `OK` в кінці | Всі тести пройшли, детермінізація коректна |
| `FAIL` | Є розбіжності — дивіться які слова не збігаються |
| `ERROR` | Алгоритм впав (crash) — помилка в коді |
| `test_cross_algorithm` — всі ok | **Головна гарантія** — всі 4 алгоритми згодні |
| `test_minimized_dfas_isomorphic` — ok | Алгоритми дають **структурно однакові** мінімальні ДКА |

## Опис тестових файлів

### test_subset.py — Subset Construction (9 тестів)

- Повертає (DFA, int) з позитивним count
- ДКА має валідну структуру
- ДКА є детермінованим
- Мовна еквівалентність для nfa_1, nfa_2, nfa_3
- Мовна еквівалентність для випадкових dense та sparse НКА
- Алфавіт збережено
- Експоненційний blowup: для n=4 "nth-from-last" НКА, ДКА має рівно 2^n = 16 станів

### test_brzozowski.py — Brzozowski (8 тестів)

- Повертає (DFA, int)
- ДКА має валідну структуру
- ДКА є детермінованим
- Мовна еквівалентність для nfa_1, nfa_2
- Мовна еквівалентність для випадкових dense та sparse НКА
- Алфавіт збережено
- Brzozowski ДКА має <= станів ніж Subset Construction ДКА (мінімальність)

### test_transset.py — Transset (8 тестів)

- Повертає (DFA, int)
- ДКА має валідну структуру
- ДКА є детермінованим
- Мовна еквівалентність для nfa_1, nfa_3
- Мовна еквівалентність для випадкових dense та sparse НКА
- Алфавіт збережено
- Коректна обробка dead state переходів (nfa_4)

### test_lazy.py — Lazy Subset Construction (10 тестів)

- Повертає (DFA, int) з позитивним count
- ДКА має валідну структуру
- ДКА є детермінованим
- Мовна еквівалентність для nfa_1, nfa_2, nfa_3
- Мовна еквівалентність для випадкових dense та sparse НКА
- Алфавіт збережено
- Розмір ДКА збігається з Subset Construction (для nth-from-last n=3,4,5)

### test_cross_algorithm.py — Крос-алгоритмова еквівалентність (8 тестів) — КЛЮЧОВИЙ

- Всі 4 алгоритми еквівалентні для predefined НКА (nfa_1..5)
- Всі 4 алгоритми еквівалентні для large НКА (nfa_large_1..5)
- Всі 4 алгоритми еквівалентні для nth-from-last(2..6)
- Всі 4 алгоритми еквівалентні для 5 random dense НКА
- Всі 4 алгоритми еквівалентні для 5 random sparse НКА
- Всі 4 алгоритми еквівалентні для multi-branch НКА
- Мінімізовані ДКА від усіх алгоритмів попарно ізоморфні
- Exhaustive перевірка (всі слова до довжини 5) — всі алгоритми згодні

### test_edge_cases.py — Граничні випадки (8 тестів)

- Один стан що приймає (з self-loops)
- Один стан що відхиляє (з self-loops)
- Немає переходів взагалі
- Всі стани приймаючі
- Недосяжні стани
- Вже детермінований НКА
- Порожні множини цілей переходів
- Self-loops на всіх станах

### test_epsilon_variants.py — Epsilon-варіанти (5 тестів)

- Всі 4 epsilon-варіанти еквівалентні для nfa_epsilon
- Обробка довгих epsilon-ланцюжків (15 станів)
- Алфавіт ДКА НЕ містить epsilon
- НКА тільки з epsilon-переходами
- Великі epsilon-ланцюжки (30 станів, 15 переходів)

### test_minimization.py — Мінімізація та ізоморфізм (6 тестів)

- Вже мінімальний ДКА залишається мінімальним
- Мінімізація зливає еквівалентні стани
- Мова зберігається після мінімізації
- Ізоморфізм рефлексивний (ДКА = ДКА)
- Ізоморфізм з перейменованими станами
- Негативний тест: різні мови — не ізоморфні

## Допоміжні модулі (helpers/)

### dfa_helpers.py

- `run_dfa(dfa, word)` — запуск слова на ДКА, повертає True/False
- `run_nfa(nfa, word)` — запуск слова на НКА з epsilon-замиканням
- `gen_random_words(alphabet, count=200, max_len=15)` — генерація випадкових слів
- `gen_systematic_words(alphabet, max_len=4)` — генерація ВСІХ слів до заданої довжини

### equivalence.py

- `check_language_equivalence_by_words(nfa, dfa)` — перевірка еквівалентності мов 300+ словами
- `check_dfa_equivalence_by_minimization(dfa1, dfa2)` — перевірка через мінімізацію + ізоморфізм
- `check_all_algorithms_equivalent(nfa, algorithms)` — комплексна перевірка всіх алгоритмів

### minimization.py

- `minimize_dfa(dfa)` — мінімізація ДКА алгоритмом Хопкрофта (partition refinement)

### isomorphism.py

- `are_isomorphic(dfa1, dfa2)` — перевірка ізоморфізму двох ДКА через BFS-біекцію
