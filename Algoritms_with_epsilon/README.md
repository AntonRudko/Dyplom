# Algoritms_with_epsilon — детермінізація НКА з e-переходами

## Навіщо потрібна окрема папка

Базові алгоритми в `Algoritms/` працюють тільки з НКА без e-переходів: кожен перехід має вигляд `(стан, символ) -> {стани}`, де символ — це реальний символ алфавіту.

Однак у теорії автоматів часто зустрічаються НКА з **e-переходами** — переходами, що виконуються без читання символу. Такі переходи записуються як `(стан, "") -> {стани}`. Вони дозволяють компактніше описувати регулярні мови, але потребують додаткової обробки при детермінізації.

Замість модифікації базових алгоритмів (що ускладнило б їхній код та порушило чистоту реалізації), e-варіанти винесені в окрему папку. Це дозволяє:

- **Порівнювати продуктивність** базових та e-варіантів на однакових тестах (Test 6 — epsilon overhead)
- **Тестувати незалежно** — базові алгоритми перевіряються без e-логіки, а e-варіанти додатково перевіряються на НКА з e-переходами
- **Зберігати простоту** базових алгоритмів для навчального розуміння

## epsilon_closure.py — e-замикання

Ключовий модуль, який використовується всіма e-варіантами.

Дві функції:

### `build_epsilon_table(states, transitions) -> dict`

Попередньо обчислює e-замикання для всіх станів. Повертає `{state: frozenset(досяжні через e)}`. Складність: O(n^3) по часу, O(n^2) по пам'яті.

### `epsilon_closure(states, transitions, table=None) -> frozenset`

Обчислює e-замикання множини станів. Якщо `table` надано — використовує об'єднання за O(n). Інакше — BFS з нуля за O(n + e_epsilon).

Приклад: якщо `q0 --e--> q1 --e--> q2`, то `epsilon_closure({q0}) = {q0, q1, q2}`.

## sub_set_epsilon.py — Subset Construction з e-замиканням

**Функція:** `determinize_nfa_epsilon(nfa) -> (DFA, subsets_processed)`

Класична побудова підмножин, адаптована для e-переходів. Відмінності від базового `sub_set.py`:

1. **Стартовий стан** — не просто `{start}`, а `e-closure({start})`. Якщо стартовий стан має e-перехід до приймаючого стану, початковий стан ДКА одразу стає приймаючим.
2. **Переходи** — після збору цільових станів за символом, до результату застосовується `e-closure`. Тобто `d'(S, a) = e-closure(U{d(s, a) | s in S})`.
3. **Алфавіт ДКА** — не містить `""` (e фільтрується).

Метрика: `subsets_processed` — кількість оброблених підмножин.

## brzozowski_epsilon.py — Brzozowski з e-замиканням

**Функція:** `determinize_brz_epsilon(nfa) -> (DFA, total_iterations)`

Алгоритм подвійного обернення: НКА -> reverse -> determinize -> reverse -> determinize. Адаптація для e-переходів:

1. **Конвертація НКА** — стартові стани = `e-closure({start})`, e-переходи видаляються зі словника переходів.
2. **e-замикання переходів** — перед першим обернення кожен перехід `(p, a) -> dsts` розширюється: для кожного `d in dsts` додається `e-closure({d})`. Це "вбудовує" e-переходи у звичайні переходи.
3. **Решта** — два цикли reverse + determinize працюють як у базовому Brzozowski, вже без e-переходів.

Метрика: `total_iterations` — сумарна кількість ітерацій обох етапів детермінізації.

---

## transset_epsilon.py — Transset з e-замиканням

**Функція:** `determinize_transset_epsilon(nfa) -> (DFA, merges_count)`

Побудова ДКА за поведінковими сигнатурами (transset = множина вихідних переходів + прапорець прийняття). Адаптація для e-переходів:

1. **Стартовий стан** — поведінкова сигнатура обчислюється не для одного стану, а для `e-closure({start})`. Функція `make_tuple_from_states(state_set)` об'єднує транзиції та прапорці прийняття всіх станів із замикання.
2. **Переходи** — після знаходження цільових станів за символом, до них застосовується `e-closure`, і поведінкова сигнатура обчислюється для всієї замкнутої множини.
3. **e-переходи ігноруються** при побудові `trans_by_state` (фільтр `if a == ""`).

Метрика: `merges_count` — кількість унікальних поведінкових сигнатур (станів ДКА).

---

## lazy_subset_epsilon.py — Lazy Subset з e-замиканням

**Функція:** `determinize_lazy_epsilon(nfa) -> (DFA, transitions_computed)`

Лінива (on-demand) побудова ДКА через кешовані переходи. Адаптація для e-переходів:

1. **Стартовий стан** — `e-closure({start})`.
2. **Ліниві переходи** — функція `lazy_move(subset, symbol)` після збору цільових станів застосовує `e-closure` і кешує результат.
3. **Матеріалізація** — DFS по досяжних підмножинах, кожна вже є e-замкнутою.

Метрика: `transitions_computed` — кількість обчислених (кешованих) переходів.

## Спільний патерн адаптації

Кожен e-варіант відрізняється від базового алгоритму у **трьох точках**:

| Точка | Базовий | e-варіант |
|-------|---------|-----------|
| Стартовий стан | `{start}` або `start` | `e-closure({start})` |
| Обчислення переходу | `U{d(s, a) \| s in S}` | `e-closure(U{d(s, a) \| s in S})` |
| Алфавіт ДКА | `nfa.alphabet` | `nfa.alphabet \ {""}` |

Решта логіки (BFS/DFS обхід, реєстрація станів, побудова ДКА) залишається ідентичною базовому алгоритму.

## Інтерфейс

Всі функції мають однаковий інтерфейс:

```python
def determinize_*_epsilon(nfa) -> (DFA, int):
    """
    Приймає: NFA (з Algoritms.class_dfa_nfa)
    Повертає: (DFA, метрика_складності)
    """
```

НКА може містити e-переходи у вигляді `(стан, "") -> {стани}` у словнику `transitions`, а алфавіт — символ `""`.
