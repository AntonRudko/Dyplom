# Algoritms_with_epsilon — детермінізація НКА з ε-переходами

## Навіщо потрібна окрема папка

Базові алгоритми в `Algoritms/` працюють тільки з НКА без ε-переходів: кожен перехід має вигляд `(стан, символ) → {стани}`, де символ — це реальний символ алфавіту.

Однак у теорії автоматів часто зустрічаються НКА з **ε-переходами** — переходами, що виконуються без читання символу. Такі переходи записуються як `(стан, "") → {стани}`. Вони дозволяють компактніше описувати регулярні мови, але потребують додаткової обробки при детермінізації.

Замість модифікації базових алгоритмів (що ускладнило б їхній код та порушило чистоту реалізації), ε-варіанти винесені в окрему папку. Це дозволяє:

- **Порівнювати продуктивність** базових та ε-варіантів на однакових тестах (Test 6 — epsilon overhead)
- **Тестувати незалежно** — базові алгоритми перевіряються без ε-логіки, а ε-варіанти додатково перевіряються на НКА з ε-переходами
- **Зберігати простоту** базових алгоритмів для навчального розуміння

## epsilon_closure.py — ε-замикання

Ключовий модуль, який використовується всіма ε-варіантами.

**Функція:** `epsilon_closure(states, transitions) → frozenset`

Приймає множину станів і словник переходів НКА. Обходить усі стани, досяжні через ε-переходи (ключ `(стан, "")`), методом BFS. Повертає `frozenset` — замкнуту множину станів.

Приклад: якщо `q0 --ε--> q1 --ε--> q2`, то `epsilon_closure({q0}) = {q0, q1, q2}`.

---

## sub_set_epsilon.py — Subset Construction з ε-замиканням

**Функція:** `determinize_nfa_epsilon(nfa) → (DFA, subsets_processed)`

Класична побудова підмножин, адаптована для ε-переходів. Відмінності від базового `sub_set.py`:

1. **Стартовий стан** — не просто `{start}`, а `ε-closure({start})`. Якщо стартовий стан має ε-перехід до приймаючого стану, початковий стан ДКА одразу стає приймаючим.
2. **Переходи** — після збору цільових станів за символом, до результату застосовується `ε-closure`. Тобто `δ'(S, a) = ε-closure(⋃{δ(s, a) | s ∈ S})`.
3. **Алфавіт ДКА** — не містить `""` (ε фільтрується).

Метрика: `subsets_processed` — кількість оброблених підмножин.

## brzozowski_epsilon.py — Brzozowski з ε-замиканням

**Функція:** `determinize_brz_epsilon(nfa) → (DFA, total_iterations)`

Алгоритм подвійного обернення: НКА → reverse → determinize → reverse → determinize. Адаптація для ε-переходів:

1. **Конвертація НКА** — стартові стани = `ε-closure({start})`, ε-переходи видаляються зі словника переходів.
2. **ε-замикання переходів** — перед першим обернення кожен перехід `(p, a) → dsts` розширюється: для кожного `d ∈ dsts` додається `ε-closure({d})`. Це «вбудовує» ε-переходи у звичайні переходи.
3. **Решта** — два цикли reverse + determinize працюють як у базовому Brzozowski, вже без ε-переходів.

Метрика: `total_iterations` — сумарна кількість ітерацій обох етапів детермінізації.

---

## transset_epsilon.py — Transset з ε-замиканням

**Функція:** `determinize_transset_epsilon(nfa) → (DFA, merges_count)`

Побудова ДКА за поведінковими сигнатурами (transset = множина вихідних переходів + прапорець прийняття). Адаптація для ε-переходів:

1. **Стартовий стан** — поведінкова сигнатура обчислюється не для одного стану, а для `ε-closure({start})`. Функція `make_tuple_from_states(state_set)` об'єднує транзиції та прапорці прийняття всіх станів із замикання.
2. **Переходи** — після знаходження цільових станів за символом, до них застосовується `ε-closure`, і поведінкова сигнатура обчислюється для всієї замкнутої множини.
3. **ε-переходи ігноруються** при побудові `trans_by_state` (фільтр `if a == ""`).

Метрика: `merges_count` — кількість унікальних поведінкових сигнатур (станів ДКА).

---

## lazy_subset_epsilon.py — Lazy Subset з ε-замиканням

**Функція:** `determinize_lazy_epsilon(nfa) → (DFA, transitions_computed)`

Лінива (on-demand) побудова ДКА через кешовані переходи. Адаптація для ε-переходів:

1. **Стартовий стан** — `ε-closure({start})`.
2. **Ліниві переходи** — функція `lazy_move(subset, symbol)` після збору цільових станів застосовує `ε-closure` і кешує результат.
3. **Матеріалізація** — DFS по досяжних підмножинах, кожна вже є ε-замкнутою.

Метрика: `transitions_computed` — кількість обчислених (кешованих) переходів.

## Спільний патерн адаптації

Кожен ε-варіант відрізняється від базового алгоритму у **трьох точках**:

| Точка | Базовий | ε-варіант |
|-------|---------|-----------|
| Стартовий стан | `{start}` або `start` | `ε-closure({start})` |
| Обчислення переходу | `⋃{δ(s, a) \| s ∈ S}` | `ε-closure(⋃{δ(s, a) \| s ∈ S})` |
| Алфавіт ДКА | `nfa.alphabet` | `nfa.alphabet \ {""}`  |

Решта логіки (BFS/DFS обхід, реєстрація станів, побудова ДКА) залишається ідентичною базовому алгоритму.

## Інтерфейс

Всі функції мають однаковий інтерфейс:

```python
def determinize_*_epsilon(nfa) -> (DFA, int):
    """
    Приймає: NFA (з Algoritms.class_dfa_nfa)
    Повертає: (DFA, метрика_складності)
    """
```

НКА може містити ε-переходи у вигляді `(стан, "") → {стани}` у словнику `transitions`, а алфавіт — символ `""`.
