# Analize — інструменти аналізу та візуалізації

Папка містить допоміжні модулі для візуалізації автоматів, перевірки коректності на конкретних словах, мокові дані та простий консольний бенчмарк.

---

## Структура

```
Analize/
├── mocks/
│   ├── nfa.py            — предвизначені тестові НКА
│   └── words.py          — набори тестових слів
├── tools/
│   ├── word_check.py     — ручна перевірка слів на НКА/ДКА
│   ├── graph_visualizer.py — візуалізація графів автоматів (GraphViz)
│   └── table_visualizer.py — візуалізація таблиць переходів (matplotlib)
└── simple_benchmark.py   — бенчмарк на випадкових НКА (консольний)
```

---

## mocks/nfa.py — предвизначені тестові НКА

Містить НКА різного розміру та складності для тестування та демонстрації:

| НКА | Станів | Алфавіт | Призначення |
|-----|--------|---------|-------------|
| `nfa_1` | 4 | {a, n, p, r} | Базовий тест з порожніми переходами |
| `nfa_2` | 4 | {e, o, t} | Множинні приймаючі стани |
| `nfa_3` | 6 | {d, k} | Бінарний алфавіт, багато станів |
| `nfa_4` | 4 | {j, k, t, w} | Великий алфавіт, dead-end переходи |
| `nfa_5` | 4 | {b, l, s, v} | Перехресні переходи |
| `nfa_large_1..5` | 6–8 | різні | Більші НКА для навантажувального тестування |
| `nfa_test1` | 4 | {0, 1} | Класичний бінарний НКА |
| `nfa_test2` | 4 | {0, 1, 2, 3} | НКА з 4-символьним алфавітом |
| `nfa_epsilon` | 3 | {a, b} + ε | НКА з ε-переходами |
| `nfa_15_states` | 15 | {0, 1, 2, 3} | Великий НКА для стрес-тестів |

## mocks/words.py — набори тестових слів

Статичні списки слів для ручної перевірки:

- `words` — слова над алфавітом {0, 1, 2, 3} довжиною 7–8
- `words_1` — слова над {0, 1} довжиною 4–6
- `words_2` — слова над {0, 1, 2} довжиною 5
- `words_3` — слова над {0, 1, 2, 4} довжиною 6
- `words_4` — слова над {0, 1, 2, 4, 5} довжиною 6
- `words_5` — слова над {0, 1, 2, 4, 5, 6} довжиною 6

## tools/word_check.py — ручна перевірка слів

Три функції для прямого запуску автоматів на словах:

- `run_dfa(dfa, word)` — запуск слова на ДКА
- `run_nfa(nfa, word)` — запуск слова на НКА (без ε-підтримки)
- `run_words(nfa, dfa, words)` — порівняння НКА та ДКА на списку слів з виводом у консоль

## simple_benchmark.py — консольний бенчмарк

Генерує випадкові НКА різного розміру (50, 100, 200, 400, 500 станів) і вимірює середній час детермінізації для кожного з 4 алгоритмів. Результат виводиться у консоль.

Параметри генерації:
- Алфавіт: {0, 1, 2, 4}
- Гарантія зв'язності (spine-ланцюг) та недетермінізму
- Ймовірність ребра: 0.15, ймовірність приймаючого стану: 0.3

```bash
python -m Analize.simple_benchmark
```

## tools/graph_visualizer.py — графи автоматів (GraphViz)

**Функція:** `visualize_automaton_graph(automaton, filename)`

Будує граф автомата (НКА або ДКА) у форматі PNG через бібліотеку GraphViz:

- Стартовий стан — невидима точка зі стрілкою
- Приймаючі стани — подвійне коло (зелений фон)
- Звичайні стани — одинарне коло (блакитний фон)
- Ребра між однаковими парами станів об'єднуються (наприклад, `a, b` замість двох окремих стрілок)
- ε-переходи відображаються як `ε`

---

## tools/table_visualizer.py — таблиці переходів (matplotlib)

**Функція:** `visualize_transition_table(automaton, filename)`

Будує таблицю переходів автомата у форматі PNG:

- Рядки — стани (стартовий позначається `→`, приймаючі — `*`)
- Стовпці — символи алфавіту (ε-переходи першими)
- Порожні переходи — `∅`, відсутні — `–`

## Heatmap-модулі

Модулі побудови heatmap перенесені в `Tests_Diagram/`:

```bash
python -m Tests_Diagram.heatmap_comparison
python -m Tests_Diagram.heatmap_algorithmic
```

### Параметри бенчмарку

- Алгоритми: Subset, Brzozowski, Transset, Lazy
- Щільності переходів: 0.02, 0.04, 0.06, 0.08, 0.10, 0.12, 0.15, 0.20
- Розміри НКА: 20, 40, 60, 80, 100 станів
- Алфавіт: {0, 1, 2}
- Для кожної точки: 5 випадкових НКА × 5 повторів = 25 вимірювань

## Опис згенерованих heatmap

Всі зображення зберігаються у `Tests_Diagram/Hitmaps/`.

### 1. heatmap_time_by_density.png

**Абсолютний час × щільність переходів** (розмір НКА фіксований = 100 станів)

- Рядки: 4 алгоритми
- Стовпці: 8 рівнів щільності (0.02 – 0.20)
- Колір: зелений = швидко, червоний = повільно
- У кожній клітинці: час у секундах

**Що показує:** як зростає час кожного алгоритму при збільшенні щільності переходів НКА. Дозволяє побачити, який алгоритм найкраще справляється з густими НКА, а який — деградує.

### 2. heatmap_normalized_by_density.png

**Відносний час × щільність** (нормалізований, 1.0 = найшвидший)

- Та ж структура, але кожен стовпець нормалізований: найшвидший алгоритм = 1.0x, решта — кратність відставання
- У клітинках: значення вигляду `1.00x`, `1.42x`, `2.31x`

**Що показує:** у скільки разів кожен алгоритм повільніший за найкращий при кожній щільності. Дозволяє порівнювати алгоритми незалежно від абсолютного часу.

### 3. heatmap_speedup_vs_subset.png

**Прискорення відносно Subset Construction × щільність**

- Рядки: Brzozowski, Transset, Lazy (без Subset — він базовий)
- Стовпці: рівні щільності
- Колір: зелений = швидший за Subset, червоний = повільніший
- У клітинках: відсоток прискорення (наприклад, `+15.3%` або `-8.2%`)

**Що показує:** на скільки відсотків кожен алгоритм швидший або повільніший за Subset Construction при різних щільностях. Subset Construction — найпоширеніший алгоритм, тому це ключове порівняння.

### 4. heatmap_time_by_size.png

**Абсолютний час × розмір НКА** (щільність фіксована = 0.08)

- Рядки: 4 алгоритми
- Стовпці: 5 розмірів НКА (20, 40, 60, 80, 100 станів)
- Колір: зелений = швидко, червоний = повільно

**Що показує:** як масштабується кожен алгоритм при зростанні розміру НКА. Дозволяє оцінити складність алгоритму на практиці — лінійна, поліноміальна чи експоненційна поведінка.

### 5. heatmap_normalized_by_size.png

**Відносний час × розмір НКА** (нормалізований)

- Аналогічно до `heatmap_normalized_by_density`, але по осі X — розміри НКА
- У клітинках: кратність відставання від найшвидшого

**Що показує:** чи зберігається перевага одного алгоритму над іншими при різних розмірах, чи відбувається зміна лідера.

### 6. heatmap_pairwise_comparison.png

**Попарне порівняння алгоритмів** (матриця 4×4)

- Рядки та стовпці: всі 4 алгоритми
- Значення клітинки [i, j]: на скільки % алгоритм i швидший за алгоритм j (усереднено по всіх щільностях)
- Колір: зелений = рядок швидший за стовпець, червоний = повільніший
- Діагональ: `—` (порівняння із самим собою)

**Що показує:** зведену картину переваг між алгоритмами. Наприклад, якщо клітинка Lazy/Subset = `+20.3%`, то Lazy в середньому на 20.3% швидший за Subset.

### 7. heatmap_grid_per_algorithm.png

**Повна сітка: розмір × щільність для кожного алгоритму** (4 субграфіки)

- Для кожного алгоритму — окрема теплова карта
- Рядки: розміри НКА (20–100)
- Стовпці: щільності (0.02–0.20)
- Колір: зелений = швидко, червоний = повільно

**Що показує:** повну двовимірну карту продуктивності кожного алгоритму. Дозволяє побачити «гарячі зони» — комбінації розміру та щільності, де алгоритм працює найповільніше, і «холодні зони» — де він ефективний.
